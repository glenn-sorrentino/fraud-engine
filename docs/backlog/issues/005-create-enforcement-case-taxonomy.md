# Issue 005: Create Enforcement Case Taxonomy

Labels: `mvp`, `research`, `taxonomy`, `priority:medium`

## User Story

As a product team, we need known FCA enforcement examples classified by fraud theory so that MVP signals can be backtested against real matters.

## Scope

Create a structured taxonomy of aerospace and defense FCA enforcement examples from DOJ public releases.

## Acceptance Criteria

- Taxonomy includes at least 15 relevant aerospace/defense procurement FCA matters.
- Each case includes defendant, date, agency, program/platform when available, alleged conduct, settlement amount, fraud theory, and source URL.
- Cases are mapped to the three MVP signal families.
- Cases distinguish allegations from admissions or determinations.
- Taxonomy can be loaded by the evaluation harness.

## Dependencies

- None.

## Notes

Initial seed examples should include defective pricing, improper markups, nonconforming parts, falsified testing, and cybersecurity misrepresentation.
